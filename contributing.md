# Contributing to Flash ⚡

Flash is open-source Bitcoin banking software. We welcome contributors worldwide — and we pay in Bitcoin.

## Bounty Program

Flash uses [Lightning Bounties](https://app.lightningbounties.com) to reward contributors with instant Bitcoin payments via the Lightning Network.

### How to Participate

1. **Find a bounty** — browse `bounty:open` labeled issues in any lnflash repo
2. **Signal intent** — comment on the issue before starting work
3. **Fork & branch** from `main` in the relevant repo
4. **Submit a PR** referencing the bounty issue (`closes #N`)
5. **Get paid** — instant Lightning payment after your PR is merged

### Bounty Tiers

- 🟢 **Starter** — 5K–25K sats — good first issues, documentation, small fixes
- 🟡 **Standard** — 25K–100K sats — features, bug fixes, integrations
- 🔴 **Prime** — 100K–500K sats — significant features, architecture work
- 💎 **Legend** — 500K+ sats — major contributions, protocol work

### Active Repos

- [flash](https://github.com/lnflash/flash) — Core backend (Galoy-based, TypeScript)
- [flash-mobile](https://github.com/lnflash/flash-mobile) — iOS/Android wallet (React Native)
- [flash-pos](https://github.com/lnflash/flash-pos) — Point of Sale & Rewards app
- [flash-pay](https://github.com/lnflash/flash-pay) — Web payment pages
- [btcpayserver-flash-plugin](https://github.com/lnflash/btcpayserver-flash-plugin) — BTCPay plugin

## Code Standards

- TypeScript preferred
- Tests required for new features
- Follow existing code style and conventions
- Reference issues in PR titles and descriptions
- Keep PRs focused — one issue per PR

## Getting Help

- Join our community and ask questions on Discord
- Use structured issue templates for bugs and features
- Tag `@lnflash/core` for review requests
